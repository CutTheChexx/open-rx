# Contributing to Open RX

Open RX is maintained by CutTheChexx. This isn't an open-contribution project — but feedback makes it better.

## How to Help

**Report issues:** Open a GitHub issue if a skill has outdated info, broken patterns, or could be improved.

**Request skills:** Want a skill that doesn't exist? Open an issue with the title `[Skill Request] your-skill-name` and describe what it should cover.

**Share results:** Built something with Open RX? Tag @CutTheChexx. Real-world usage drives what gets updated first.

## Update Schedule

Open RX updates on the **1st of every month**. Priority goes to:

1. Breaking changes in dependencies (Next.js, Tailwind, GSAP, Supabase, Stripe)
2. New patterns from top open-source repos
3. Community-requested improvements
4. New skill development

## What Won't Be Accepted

- PRs that add skills (skills are authored by CutTheChexx only)
- PRs that modify the license or attribution
- PRs that add personal branding or company-specific patterns

## Contact

- **GitHub Issues:** Preferred for bugs and requests
- **Email:** cutthechexx@openrx.dev
